## 结论先行（一句话）

**是，我们陷入了局部最优。数学上，总账 >600k 的任何方案都不可能超过榜首 93.49（需要 K>100 键）；
而我们把 90% 的火力花在了 1.6M–3.4M 区间的精修上。唯一可行的胜利区域是
"≥96 键 @ ≤499,999 tokens（精确装填至峰顶）"，这要求把 ins/fin 两域的答题成本压缩 5–8 倍而不掉键——
现有"逐题重付证据"的架构做不到，必须换成域级单上下文联合作答 + 分诊升级。**

## 1. 理论最优测算：当前架构的天花板 = 87±1，永远够不到 93.49

### 1.1 分数模型（已实测校准）

```
S = 0.5×K + 0.3×R + 0.2×T(t)
T(t) = t/500,000×100        (t < 500k, 递增段)      ← 峰顶 499,999 ≈ 100
     = (5,000,000−t)/5M×100 (t ≥ 500k, 递减段)      ← 500,001 即断崖到 90
R 实测天花板 = 85（四段式与精简两风格同分；sim87.6 未经真值验证）
```

### 1.2 反解榜首门槛（R=85 时超过 93.49 所需键数 K）

| 总账 t | T(t) | 需要 K | 可行性 |
|---|---|---|---|
| 499,999 | ~100 | **≥96** | **唯一可行区域** |
| 600k | 88 | ≥101 | 不可能 |
| 1.0M | 80 | ≥104 | 不可能 |
| 1.6M（路由预期） | 68 | ≥109 | 不可能 |

**这是全文最重要的一行算术：t≥600k 时 K 需求超过 100，胜利在数学上被关死。**
（若 R 能到 87.6 真值，门槛降为 95 键@499k；若 R=90，94 键@499k=94.0 分。R 每+5 抵 3 键。）

### 1.3 现有零件在各区域的最优组合

| 方案 | K | t | 得分 | 备注 |
|---|---|---|---|---|
| full11 | 95 | 3.37M+68k推理 | **79.3** | 键最高，token 判死刑 |
| b_router1（今日实跑） | est 89±2 | 1.72M+68k | **82.9** | 已知键30/38；fin腿爆到1.06M |
| 路由理想执行 | 94 | 1.6M | **86.1** | 即背景里的"86-88"预期 |
| slim17+lean推理 | 77 | 492,566 | **83.7** | 键太少 |
| 域最优帧内拼装极限 | ~87 | ~575k | **86.7** | reg20@57k+res19@76k+fc18@98k+ins17@144k+fin13@200k，已超预算且键只有87 |

**当前架构理论最优 ≈ 87（无论走路由还是拼装），距榜首缺口 6.5 分 ≈ 13 个键的等价物。**
这不是执行问题，是架构问题：87 是"逐题重付证据"范式的上确界。

### 1.4 目标形式化

> **总攻目标：K ≥ 96 @ t ∈ [480k, 499,999]（含推理生成、docsel、一切），R ≥ 85 → S ≥ 93.5。**
> 当前 sub-500k 最好键数 = 77（slim17）。缺口 19 键，全部集中在 ins（13→19）、fin（11-13→18）、fc（17→19）。

### 1.5 递增段的战略含义（容易被忽略）

t<500k 段 token 分随消耗**递增**（250k 只有 T=50）。所以 **500k 不是"上限"而是"预算"——
低于 499k 的每一个省下的 token 都是白扔的分**。正确玩法是精确装填（lean99b 协议已验证）：
把预算全部花在准确率与推理质量上，停在 499k。"省token"这个词从此应从字典里删除，换成"重新分配token"。

## 2. 我们卡在哪个局部最优？——四个被固化的假设

### 假设A：「证据必须逐题重付」（最贵的一个）

- 现状：full 帧每题独立调用，fin 中位数 **49.7k tok/题**（b_router1_finfull 实测），20 题重复支付同样的
  4 份年报证据 20 次，再乘 r1×2/r2/仲裁的采样次数 → fin 一域 1.06M。
- 而全部 16 张保险记忆卡合计仅 **17.8k 字符（≈13k tok）**；fin 的 digest 全集也在 10k 级。
  共享底仓极小，重复支付极大——结构性浪费 >80%。
- 为什么固化：slim 线试过合批，但**永远是"合批+饿死"绑定出现**（瘦帽、砍思考、截证据），
  于是把"批量稀释掉键"的账记在了合批头上（规律8"广度型修复必须配预算"全部是在瘦帽下取得的证据）。
  **"满帽合批"（一个上下文装一域全部题+全部卡+全部保护块）从未被实验过。**
- cards2 的 fc12/ins8 也不构成反证：那是"有损压卡"实验，卡本身被砍掉了信息。

### 假设B：「跨代异构二审=选择题专属」（管线暗沟，26 道计算题裸奔）

- 代码事实：`answerer.py:674` 从 **env** 读 `AFAC_VERIFY_MODEL`（选择题路径生效）；
  但 `calc.answer_calc` 的 verify_model 只从 **CLI 参数** `--verify-model` 传入（`run_b2.py:78,139`），
  而 full11/router1 都只设了 env → **计算题的第二样本从来都是同模型自我确认**。
  （run_b2 头部打印 `verify=qwen3.6-plus` 也是取 CLI 参数的显示假象，掩盖了这一点。）
- 今日实锤：router1 的 fin_b_014 答 `8.70`（键 8.69）、fin_b_020 答 `112.33|112.31|0.02`（键 112.32|112.31|0.01）
  ——全是取整/精度级摇摆，正是"同模型双样本无法打破错误相关性"的教科书案例。
  full11 用跨代异构在选择题上+5 键，同样的武器在 26 道计算题上从未开火。
- 顺带：取整规则从未统一（银行家舍入 vs 四舍五入 vs 中间过程精度），应改 Decimal+显式规则。

### 假设C：「推理分贴死85、推理必须单独后置生成」

- 单独的推理生成 pass 花 68k（lean）～153k（thick），占 500k 预算的 14–30%，**却与答题 pass 完全重复劳动**
  （答题时模型本来就产出了定位/提取/推导过程，被扔掉后再花钱重新生成一遍）。
- 融合方案：答题输出直接带四段式精简推理（同一次 completion 同时供 acc 与 R 使用）→ 68k 直接归零。
- "85天花板"的证据强度有限：真值锚点只有 84.65/85 两个，且都在同一类风格族内；
  r90 冲刺失败是**生成churn**（双候选采样噪声）不是 judge 上限的证明。R 每+5 抵 3 键，这条轴未探明。

### 假设D：「解码出的事实 ≠ 运行时规则」（已付学费不回流）

- **reg_b_004 / reg_b_018 两道日期题，已解码键为数字格式 `20260630.00` / `20260401.00`，
  但今天所有在跑配置（full11/full12/slim17/cards2/router1）仍统一输出中文日期 → 每次新跑白送 2 键。**
  b_schema 的"readme日期规则升级"只被 reg_b_003 验证过，被错误地全域化了；占位符均为 999999.99，
  说明平台判卷逐题而非按 readme。类级修法：日期槽输出经平台验证过的格式族（合规注记见 §6）。
- eval/b_known_labels.json 里 `res_b_005: 22.19` 与台账"全场0命中/键未验"矛盾——已知键文件混入了未验证猜测，
  会让所有基于它的评测虚高 1 键（本文件的 38 条统计已受影响）。需要立即分级标注（verified / inferred / dead）。

## 3. 跳出方向提案

### P1 域级单上下文联合作答（"满帽mega-context"）——攻击假设A

- **做法**：每域一次（fc 可 4×5 子批）大 prompt = 该域全部记忆卡 + 逐题配额检索并集（满帽，不饿死）
  + 全部题目；输出 = 每题 {答案, 四段精简推理, 置信度, 证据充分性标记}（JSON，推理融合输出，攻假设C）。
  第二 pass 用 **qwen3.5-plus 读同一 FactPack + 一审答案做域级异构复核**（FactPack 每域只重付 1 次而非 20 次）。
- **预算测算**：FactPack fin≈45k/ins≈30k/fc≈35k/res≈25k/reg≈20k，两 pass ≈ 2×(pack+题+输出)
  → 全卷 ≈ 330–400k，含 docsel 30k 仍留 70–140k 给分诊升级。对照现状：仅 fin 一域就 1.06M。
- **预期收益**：t 从 1.7M → <500k（+13 token 分）；键数是唯一悬念。
- **风险**：①20题共场注意力稀释/互扰（slim 证据是瘦帽下取得的，满帽下未知——这正是要做的实验）；
  ②长输出截断与解析失败（需严格 JSON+修复重试，禁默认值）；③一个错误前提在批内传播（异构二审对冲）。
- **验证实验 E1（先打 fin，最贵最难的域）**：fin 20 题一个 mega-prompt（4 份年报卡+满帽块），
  两变体（单 pass / +3.5-plus 域级复核），对照 full12 的 fin18@721k，用 11 条 fin 已知键判分。
  成本 ≈150–250k tok。**通过门槛：已知键 ≥9/11 且 est fin ≥16 @ ≤200k** → 绿灯推 ins（E2，门槛 ins≥17@≤150k，
  对照 insfresh 19@671k）。两域任一不过 → 退化为"该域小批×4"再测，仍不过才放弃。

### P2 两阶段难度分诊（廉价全卷一遍 → 低置信题重装二审）——与P1合体

- **做法**：P1 的 mega-pass 即"廉价一遍"（带置信度+证据充分性自报）；
  升级触发 = ①两 pass 不一致 ②自报低置信 ③历史摇摆类清单（从57次运行学的路由规则，合规）。
  升级动作 = 单题 full 管线+跨代异构（15–25k/题），**硬上限 8–10 题**（预算闸门，防 slim12 式叠加爆仓）。
- **预期收益**：把 full11 的"每题都全价"改成"只给难题全价"。历史摇摆集稳定在 10–15 题规模，
  正好压在 150–250k 升级预算内。
- **风险**：置信度自报不可靠（需用已知键校准触发阈值）；升级题选偏 → 用"两pass不一致"这个硬信号兜底。
- **验证实验 E2b**：在 E1/E2 产物上跑分诊，统计升级集合对已知错题的召回率（目标 ≥80%）与误升率。

### P3 结构化事实表（fin/ins 数字底仓，离线词法构建）——攻击 fin 深井的病根

- **做法**：对 4 家公司年报离线抽取标准化表（营收/净利/分红中期+末期/权益/EPS/现金流…逐年），
  纯规则/版面表格恢复（合规：预处理表格恢复允许，非语义摘要，内容全为原文数字）；
  ins 同理抽"给付比例/等待期/犹豫期/免责清单"结构表。表进 FactPack 替换一半原始块。
- **预期收益**：fin 取数错误类（fin_b_014/020 精度摇摆、fin_b_005 分红两笔、fin_b_012 母公司报表）从根上消灭；
  FactPack 体积再减 30–50%；26 道计算题从"检索赌命"变"查表计算"。
- **成本**：离线工程 0.5–1 天，0 线上 token。风险：抽取错误污染（需抽样人工校验 + 保留原文块兜底）。
- **验证实验 E3**：8 条 fin 计算已知键全对 + prompt 体积下降为通过。

### P4 3.7 家族性格重校准（fin 专家 / 复核官，不当全卷主答）

- 证据：slim26/slim28 的 fin13=slim史高、full14 的 fin17一致性——3.7 家族 fin 强项跨配置复现；
  整体崩（ins8/res13）疑为 3.6 调校的解析/提示不合拍，不是能力问题。
- **做法**：只做两件窄事：①3.7-max 当 P1 的域级复核官（替代或并联 3.5-plus，比较谁的分歧检出真错多）；
  ②为 3.7 重写判分口径少样本（现口径是 3.6 的错误模式教材）后，fin 域单挑 full12 的 fin18。
- 预期 +1–2 fin 键；成本 ~300–500k 实验 token；**风险：fin 是全线方差最大域（±3），单跑信号弱，
  必须用 11 条已知键+双跑取平均判定**。优先级低于 P1–P3，仅当 E1 后 fin 仍 <16 时启动。

### P5 计算题跨代异构修通 + 取整规则统一（立即做的零成本手术）

- ①calc 链路读 env 的 AFAC_VERIFY_MODEL（或 router 补传 --verify-model）；②第二样本强制换代；
  ③Decimal + 显式取整规则（从已解码键反推平台口径：8.69/0.01/112.32 提示"中间不进位、末位截断优先"，需验证）；
  ④修 run_b2 头部 verify 显示假象（打印 env 生效值——上次 env 漏设事故的观测盲区就在这）。
- 预期：直接回收 fin_b_014/020 类 2–3 键，26 题计算全线受益。成本≈0（管线修补）+200k 复测。

### P6 res_b_005 类"无解题"的处理策略（EV 封顶，止血）

- 事实：五个小数口径全灭（22.19/21.74/21.69/22.37/22.27），活口径 22/18.05/18/26.64 未验；
  单题 EV ≤ 0.5 分，而历史上为它烧过 20k+ 特护（"知识缺失砸钱无用"已入失败清单）。
- **策略**：①运行时按题类配"花费地板"——检索置信度低于阈值的题只走 lean 单 pass（省下的进装填预算）；
  ②答案取活口径中概率最高者（全口径 18.05 的题设契合度论证最强），**但推理文本必须照常打磨**
  ——R 分只看文本不看对错，0.3 权重下一篇好推理 ≈ 0.6 个键，无解题的推理分是白捡的；
  ③已知键文件立即把它降级为 inferred，评测口径同步修正；④若还有提交解码窗口，用不占用更优探针的空档差分它。

### P7 推理分 90 探针（低优先，机会主义）

- 若有富余提交槽：一次提交里对 20 题换第三种风格（如"证据引用密度加倍+结论前置"），差分解码 R 是否可动。
- R 85→90 = +1.5 分 = 3 键等价物，且与所有其他工作正交。成本 = 一个提交槽，谨慎使用。
