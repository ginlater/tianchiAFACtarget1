## 〇、三十秒读法（接手工程师先看这里）

1. 这场比赛的 token 工程只有两个真问题：**哪些钱买到了键，哪些是零键支出**。本册第三章的
   汇率表就是答案，其余章节是它的证明。
2. 实测评分曲线下 **1 键 = 0.5 分 = 125k tokens**（第五章）。任何 >125k/键 的支出经济上是亏的，
   任何零键支出每 100k 直接烧 0.4 分。
3. 当前技术栈的系统帕累托点：**95 键 @ ~1.6M（单发自含账）**。两条独立路线（slimKing4 架构、
   分域冠军件理论拼装）收敛到同一点（第四章）。
4. 想动任何一笔账之前，先查第一章十二条定律——每条都有人用真金白银试错过了。

## 一、十二条 Token 定律

### L1. 饿死定理：瘦帧下所有错误都是证据层饿死，判断层加钱无药

- **证据**：slim17/21/22/24/25 五种判断层配方（有无卡/投票/异构）全停 77-78 键；slim23 加
  3.5-plus 异构二审 **+354,824 买 0 键**，slim29 加 3.7-max 复核 **+476k 反 -5 键**；
  同代码把证据帽 ×2.5 进 full 帧即到 90-95 键 → 约束在证据帽本身，不在判断层。
- 来源：TOKEN_SLIM T2、第四章键顶对照组。

### L2. 结构级 >> 参数级：省钱先换结构，参数刀只配打包边际

- **证据**：结构刀四连全部"省钱还增键"——合批摊薄 -91.7k/+3键（T3）、盲检批量化 -57.6k/零漂移
  （T4）、b4 截断 -64.8k/+1键（T5）、稀释闸门 -73k/+7键（T7）；参数刀单用即翻车——slim4
  无脑收紧帽+思考 **-98k 但 -5 键**（ins 10→6）。
- 来源：TOKEN_SLIM T3-T8。

### L3. 膝点定律：证据帽的可压区间只有噪声尾巴，越过即承重墙

- **证据**：EV_CAP ×1.0→×0.55（-45%），fin.r1 prompt 仅 -5.6%——帽根本管不到账的大头
  （构卡+指令+题干+保护块豁免）；×0.7 是甜点（fin 16→18 反增键）；×0.55 破膝点砍进 calc
  取数层（fin.calc1 prompt -28%、res.calc1 -45%）→ **95→90，省 125.6k 赔 5 键**（slimKing5，
  commit 0aef75b）。选择题 r1 有噪声余量，calc 取数没有——数字散在多张表，砍即断粮。
- 来源：TOKEN_SLIM 第三章。

### L4. 去噪增键：增键不是买的，是删出来的

- **证据**：帽 ×0.8→×0.7 仅 +14.5k 换 +2 键（SK2→SK3，`[核验修订]` 由错转对的是
  **fin_b_005/008**，原文误写 fin_b_004——该题两版均对）；insE 全文卡→15k 节选卡
  `[核验修订]` **~162k→110,969（省 32%）且首次 20/20**（原文 350k→118k 两端均无台账支撑，
  错数沿袭自 CLAUDE.md；方向结论不变）；finsB 4.5k 瘦上下文灭掉 full 帧混合证据里的
  近似数字诱饵（`[核验修订]` full 帧该题逐题账实为 ~35k…112k，原文"24k"无一手对应值；
  同题 full 帧 16 跑 9 错三种错法=摇摆，批量帧 8/8 全对）。"更小上下文=更少摇摆源"。
- 来源：TOKEN_SLIM 第三章、TOKEN_FULL 2.3、TOKEN_MIX 3.2。

### L5. 异构定律：唯一值钱的验证是跨代异构；同模型重采样 ×2 饱和；弱模型只探不裁

- **证据**：full9(flash r2=766,844)=90 → full11(3.5-plus r2=762,552)=95：**同价换模型 +5 键**，
  全战役最值的 763k；full10 r1×3 **+749,912 买 0 键**（4.59M 全线最贵仍 90）；full8 flash 当
  r3 法官 214,028 **倒扣 2 键**；b_final 复核通道装 3.6-max-preview 花 992,463（占账 35.4%）
  只得 84 键。
- 来源：TOKEN_FULL 1.2/1.3。

### L6. 共享真实性定律：批量只许摊共享成本，逐题边际 MSIS 一字不能摊

- **证据**：reg/res/fc（批内共享文档底仓，题差在问法）批量代价 -0~-2 键、便宜 4-10 倍；
  ins/fin（每题边际 MSIS 是特定条款/特定单元格）塌 **-5~-12 键**，且塌法是批级团灭——
  routerP 的 ins 8 题批 **整批 0/8**，mixKing 同帧同 prompt 重掷 6/8：批量把逐题独立方差改造成
  批级相关方差。批成员地板价是域常数（reg 恒 2.5k / res 3.6-3.7k / fc 4.4-4.5k，跨 4 run 不变）。
- 来源：TOKEN_MIX 1.3、2.1-2.3（含可迁移判据 checklist）。

### L7. MSIS 恰好送达定律：便宜与准同源，题的真实成本=其 MSIS 价，不是模型档次价

- **证据**：fin_b_004（速动比率，存货行与题面 2-gram 零重叠）full 帧 16 跑 7 对 9 错
  @34.9k-111.6k/题；批量帧 **4.5k/题 8/8 全对**——确定性注入（_RATIO_NEED 表）补覆盖 +
  facts2 四维绑定提密度 + 瘦上下文灭摇摆。绑定缺失用采样预算不可替代（full10 烧 4.59M 仍
  治不了 fin_b_012）。边界：这是题级现象不是域级现象，finsB 的 fin 总键面仍 11/20。
- 来源：TOKEN_MIX 第三章。

### L8. 架构代差定律：帧内减法伤键的部件，换架构后贡献归零——省大钱靠换架构

- **证据**：full 帧内去 r1 二票掉 4 键（full13，fin 18→15）；但 slimKing4 整体去掉
  r1 二票 + r2 + r3 共 **-1,923,160（占 fullKing 账 55%），键反 +1**（94→95）——换成
  "节选卡去噪+帽0.8+异构降维进 calc2" 后整个复核/投票超结构对键面贡献归零。三矿在 full 帧
  +108,203 买 -1 键（白加），在 slim 帧是承重件。slimKing 族四掷 92→92→94→95，非单次好骰。
- 来源：TOKEN_FULL 第三章。

### L9. 卡是准确率部件，不是省 token 部件

- **证据**：大帽下卡买键（slim6 保险卡 ins 6→17）；瘦帽下三试盖棺——slim15 -7 键、
  slim24 +0 键 @+250k、slim25 +1 键 @+384k（卡挤占证据帽）；full11 用缓存卡 digest=0 仍 95 键
  （迭代期 fresh 构卡每跑 19-39 万可全免）。合规提交件必须 fresh 入账时用节选构卡
  （WHOLE_LIMIT=15000）：slimKing4 全域 fresh 仅 238,126，比全文卡省 38%。
- 来源：TOKEN_SLIM T1、TOKEN_FULL 1.3/2.1。

### L10. 双件套定律：共享上下文必须 per-成员配额 + 总量闸门，缺一即回归

- **证据**：slim12 只有配额没闸门，保护豁免×8 题叠加账爆 576,359@66 键；slim13 加闸门
  **503,160@73（-73k 且 +7 键，slim 线最大单版提分）**。定量法医：8 题批检索覆盖 0.16 vs
  单题 0.41；批内覆盖 <0.15 时正确率 48%，≥0.4 达 86%。
- 来源：TOKEN_SLIM T7、TOKEN_MIX 2.2（batch.py:70-71, 102-107）。

### L11. 格式服从性定律：摊薄经济学建立在模型守块格式上，换模型先验格式

- **证据**：finsC 同帧同题面换 3.5-plus 当主答：不守【第i题】块格式 → **27 发 b4 追答
  +132,770，总账 +73%，键还更差**（19 vs 23）；slim27 换 3.5-plus 主答：b4 暴涨 333,735
  （72 次），**51 键**——帧/解析器按 3.6 整定，换代=兜底雪崩。
- 来源：TOKEN_MIX 1.3-5、TOKEN_SLIM 雷表。

### L12. 账实生死线：逐题账必须真实、自含、可复算——合成账期望为负

- **证据**：router6-v2 三行合成账 [4989,880] **全库扫描 0 命中**、低报 11,996，按真账重算
  >500k 峰顶不复存在，被查实属取消资格级；而诚实真件替换只 +4,706 仍可回填峰顶带——
  **合成账完全没有必要**。cards2 首题记账 57 题零账=审计红旗（新规明文"逐题 token 须与题长
  相关"）。正确协议四条：调用时均摊（余数入伪键防双计）/ 伪键全保留进 totals / 提交时
  divmod 摊平保证逐题合计==summary / resume 合并历史账。缓存吃账须申报来源。
- 来源：HEX_OBSERVER 第 2/5 节、TOKEN_MIX 第四章。

## 五、与评分曲线的耦合

### 5.1 实测曲线（HEX_OBSERVER 闭环解码，唯一实测锚点）

- b_hex 总账 577,459 → T=(5,000,000−577,459)/5,000,000×100=**88.4508**，与线上分解码
  **92.6902 = 99×0.5 + 85×0.3 + 88.45×0.2** 完全吻合。两个实锤：
  1. **≥500k 段平台用旧公式**（公告"[500k,5M]=满分100"已被证伪，公告规则≠平台行为）；
  2. **实测权重 = acc×0.5 + 推理×0.3 + token×0.2**（公告 0.6/0.2/0.2 ≠ 平台）。
- <500k 段**无实测点**。T(499,122) 三情景：两段假设成立 99.82（+2.27 分）/ 旧公式全域
  90.02（+0.31）/ 90×tot/500k 读法 89.84（+0.28）——上行 +2.27 依赖未证实段，下行保底 +0.28。

### 5.2 换算表（≥500k 段，实测权重）

| 量 | 分值 |
|---|---|
| 1 键（1 个百分点 acc） | **0.5 分** |
| 100k tokens | **0.4 分**（ΔT=2 × 0.2） |
| 1 推理分 | 0.3 分（权重最重的隐藏项：推理 85 定格贡献 25.5 分） |
| **键-token 盈亏线** | **125k/键**（第三章汇率表的标尺） |

若 <500k 线性读法成立，峰顶左侧 100k = 4 分——比右侧陡 10 倍。**峰顶站位纪律**：只有总账
已在 500-600k 量级才谈峰顶（b_hex 577k → 站 499.0-499.9k 带，期望 +0.28~+2.27）；对 1.6M
量级的单发合规线，峰顶不可达也不必达，专心用 0.4 分/100k 与 0.5 分/键做交换。
**且任何站位工程以账实为前提**：靠合成账站上 499,122 被查实是取消资格级，期望值为负（L12）。

### 5.4 作战结论

1. 提准确率靠跨代异构（放在 calc2 位，19.5k/键），省 token 靠换架构（slimKing 骨架），

   （空或 <20 字=0，模板化被明文严审）。
