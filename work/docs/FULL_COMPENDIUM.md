### 1.1 LEAN_R2（AFAC_LEAN_R2=1）——复核瘦证据

- **掉多少**：整包-2键（95→93）。但同为r1×1的严格对照 full12(93) vs full13(91)：**瘦证据r2不但没掉、还+2**——保护集+引用页已覆盖复核所需信息，全量证据在r2里是纯冗余。

## 二、提分小代价（逐条：操作/+几键/花多少/机理）

### 2.1 跨代异构二审 = 全战役最强单一武器（+5键，零增账）
- **操作**：AFAC_VERIFY_MODEL=qwen3.5-plus，r2换老一代模型独立复核（主答保持3.6-plus）。
- **+几键/花多少**：full9→full11 同骨架，r2通道flash 766,844 → 3.5-plus 762,552（**成本持平**），键90→95（**+5**）。LEAN档只需634,870（full12）。

## 三、权衡定理

### 3.1 full的错误模式 = 判断摇摆（实证定义）
14次full运行（full2-15）错误频次分布呈明显双峰：
- **摇摆带**（同代码重跑随机翻面）：res_b_013(9/14错)、fin_b_016(9/14)、fin_b_004(8/14)、fin_b_019(5/14)、fin_b_014(5/14)、ins_b_009(4/14)、res_b_014(4/14)、fc_b_003(3/14)、fin_b_020(3/14)、res_b_012(3/14)。
- **顽固带**（结构性缺失，投多少票都不动）：res_b_008(14/14错)、res_b_005(14/14)、fin_b_005(13/14)、fin_b_012(13/14)、ins_b_010(11/14)。

**定理**：投票/异构武器只治摇摆带（90→95的+5全部来自摇摆带收编）；顽固带是检索/知识缺失，与slim的"证据饿死"同源，判断层武器无效（slim23异构实证无增益）。诊断方法：跨≥3次运行数翻面次数，翻面≥2次→上投票；从不翻面且一直错→查检索。

### 3.2 键-token弹性曲线（三段式）
```
84@2.81M(b_final) → 89@2.61M(full2) → 90@2.80M(full3)      【第一段:类修复段】
→ 87-90@2.33M-4.59M(full4-10七连,含full10烧4.59M仍90)      【第二段:同模型平台】
→ 95@3.37M(full11) / 93@2.35M(full12) / 91@2.56M(full13)   【第三段:异构段】
```
- **第一段免费**：+6键靠判分规则/检索类修复，token不增反降。
- **第二段加钱无用**：full10比full4多花2.27M只多3键且未破90——同模型单跑天花板=90±1，五连87-90实证。
- **第三段结构性**：破平台唯一路径是换错误分布（跨代异构），不是加预算。
- **异构后有效前沿**：full12 93@2.35M ↔ full11 95@3.37M，边际键价≈506k/键（r1×2的764k买4键+LEAN差额）。
