## 3. 病理机制（模型自己招供）

三条独立证据链指向同一机制——**选项级证据在共享上下文中被配额挤掉 →
"证据不足不选"纪律 → 系统性漏选**：

1. **c1 原话**。批量记录的 c1 文本中大量出现"证据中未提供/未提及/无法核实"措辞：
   - 美的母公司批（fin_b_005/008/011/012 同批）：33/36 条记录 c1 含
     "证据中未提供美的集团母公司营业收入数据，无法判断。A不选。"——正是 fin_facts2
     根因（合并/母公司列身份）在批通道的重演；
   - ins 各批 18~22/34 条含"XX条款在证据中未提及→不选"。
2. **b_slim20 铁案**（完整证据日志 n_ev=8，无截断）：同质批 [ins_b_002,004,014]（docs=1,15,16,2）
   证据分布 `{1:5, 15:1, 16:2, 2:0}` ——**doc 2（国寿增益宝）0 块**，c1 原话
   "国寿增益宝条款未在提供证据中，无法核实……暂不选"，ins_b_002/004 双双答成单字母。
3. **ins 核心题全是"哪些产品明确列明X"型跨产品存在性题**（一个选项=一份文档：
   未成年人限制/自杀条款/核风险/地震免责/年龄错误/减额交清）。域级松散合批 union=10+ 文档、
   每题配额 1/7~1/8，单个产品的特定条款极易整块缺席 → 该选项被"负证据"掉。

r1/r2 佐证：核心题 277 条错误记录中 **262 条 r1 即错且 r2 缺失**（slim 批砍掉了复核轮）；
对照 b_v4（strict 批 + 异构 r2）fin_b_004 被 r2 从 A 纠成 AB——**异构二审在批通道同样是救命件**。

## 5. 剂量反应：异底仓程度 → 错误率

expansion = |批文档并集| − |本题文档数|：

| fin expansion | 正确率 |   | ins expansion | 正确率 |
|---|---|---|---|---|
| 0（纯同质） | **32/38 = 84%** |   | 0 | 0/2（仅 b_slim20 严帽铁案，见§3） |
| 1–2 | 34/67 = 51% |   | 3–5 | 25/52 = 48% |
| 3–5 | 147/260 = 57% |   | 6–9 | 155/285 = 54% |
| 6–9 | 36/98 = 37% |   | 10+ | 81/176 = 46% |

同题内对照（fin）：fin_b_003 同质批 **6/6** vs 异底仓 8/35=23%；fin_b_009 **6/6** vs 13/36=36%；
fin_b_011 exp≤2 23/30=77% vs exp≥3 2/6=33%。fin 批大小 2（严格同 docset 对）86.8% vs 批 8 49.6%。
**b_v4（可信键 run）本身就是 strict 同质批 fin 三对 + 异构 r2 → 12/12。**
ins 几乎没有同质批对照样本（docsel 出的 docset 彼此不同），剂量曲线平坦——ins 的伤害不随
expansion 单调，因为**任何 expansion>0 就足以挤掉某产品的关键条款**（阈值效应，非线性）。

（批内位置效应 fin pos4/5 只有 6%/19% 正确——但位置与 qid 绑定（按 docset 排序分批），
与题目难度混杂，不作独立结论。）

## 6. 处方一验证：同质批组 `_group_homo`（docs(B)⊆docs(A) 才并）

用 b_routerP / b_slim24 / b_mixKing 三个 run 的真实 docsel picks 离线模拟，结构稳定一致：

- **27 题 → 20 组：16 题单飞 + fin[003,009] 对 + fin[011,012] 对 + ins[002,004,014] 三题组 +
  fin[001,004,006,010] 四题组**；expansion 从旧 2–13 降到 0–3。
- 11 道团灭核心题里 8 道变单飞（直接回到 solo 通道，solo 率 82–100%），
  fin_b_003/009 落入已被 6 个 strict run 证明 6/6 的同质对。
- **预期净救：homo 预期 21.1~22.4/27 vs loose 实测均值 13.6 → +7.4~+8.7 键**
  （保守档对合批成员打 ×0.85 折）。

**判定：对症。** fin 有直接实验证据（strict 批 84%/纯对 87%/b_v4 12/12）；ins 是机制外推
（单飞=solo 通道，solo 率实测），方向确定、幅度按 solo 率可信。

残余风险（homo 不是全保险）：
1. **fin_b_006 的子集并入陷阱**：picks={cmb25, byd24}（byd24 是 docsel 强制≥2 文档凑的假文档），
   若 fin_b_010 的 picks 恰含 cmb25（docsel 方差）则 006 被并进 byd/catl 四题组（exp=3），
   招行 KPI 仍要与 3 道 byd/catl 题抢配额——它 solo 100%、批 6%，值得单飞白名单或剔除凑数文档后再分组。
2. **b_slim20 铁案证明：同质批 + 严帽（SLIM4 per_cap）照样饿死整份文档**。homo 必须配合
   足够的每文档证据底仓（ins 卡帽 8000 在 3 题批下 doc2 仍为 0 块）——并集零膨胀解决"宽"，
   不解决"帽太小"。
3. ins 三题组唯一实测样本 1/3 正确（严帽下），组内仍建议保 4 题以下+帽随文档数扩容。

## 9. 结论

1. **团灭病根确认，但比 mix 复盘假设更细一层**：不是 doc 级"装不下"（饿死仅 0.2%），
   而是**并集配额把选项粒度的条款/单元格挤出上下文，撞上"证据不足不选"纪律 → 漏选字母**。
   ins 的跨产品存在性题型（一选项一文档）是完美受害者。
2. **同质批处方对症**：fin 有 6/6、84% vs 37–57%、b_v4 12/12 三重实证；模拟显示 27→20 组、
   8/11 核心题回归 solo 通道，预期 +7.4~8.7 键。
3. **三矿入批处方对症且互补**：fin_b_012、ins_b_010 有 6/6 vs 2–4/16 族级铁证；
   但有三个矿洞（地震/核/银行KPI）恰好对应 ins_b_012、ins_b_017、fin_b_006——
   这三题在 homo 结构里全是单飞，可由 solo 通道兜住（solo 92%/97%/100%）。
4. **第三杠杆勿忘**：批通道保留异构 r2（262/277 核心错是 r1 错且无 r2；b_v4 的 r2 实际救过
   fin_b_004）；以及严帽是同质批下唯一复现过的死法（b_slim20），帽要随批内文档数扩容。
5. 剩余硬损：fin_b_005（全系统 3%）、fin_b_004（骰子）≈ 1.5–2 键，非批量问题。
