# KS_MECHANISM — 知识结构的胜负机制分析

> 编成 2026-07-25 前夜。材料：docs/ 四册手册、MIX_LOG/FULL_LOG/SLIM_LOG、FAMILY_DOCTRINE、
> GLOBAL_ESCAPE、各 run 的 token_ledger/answers（finsB 的 fin_b_004=AB@4,493tok 已直接核账）。
> 目的：不是复盘"哪个配置赢了"，而是回答**赢的知识结构为什么赢、输的为什么输**，
> 提炼可带出本赛题的设计原理。

---

## 0. 分析框架：最小充分信息集（MSIS）与四类信息

**MSIS（Minimal Sufficient Information Set）**：对一道题，模型答对所需进入上下文的
原子事实集合——且每个事实必须带着完整的**绑定坐标**（数值↔实体↔口径↔期间↔出处）。

一个知识结构的价值 = 它以多低的成本、多高的保真度把 MSIS 送进上下文。
按"补的是什么类型的信息"分四类：

| 类型 | 定义 | 缺失时的病征 |
|---|---|---|
| **数值绑定** | 数字与其列身份/口径/期间的耦合 | 数字汤：数值在场但归属不明（合并vs母公司） |
| **身份消歧** | 证据块与实体（公司/产品/文档）的归属 | 张冠李戴：拿A产品条款答B产品题 |
| **覆盖保障** | MSIS 成员一个不漏地在场 | 饿死：漏检的成员无法被任何判断层挽救 |
| **表达对齐** | 题面词汇/答案格式 ↔ 文档词汇/判分口径 | 隔墙：证据在库里但检索/判分够不着 |

胜负判据（本文全部案例可归结为一句）：
**赢的结构恰好补上缺失的那一类信息且几乎零体积；输的结构用体积或采样次数
去替代缺失的绑定/覆盖，或让一个未验证覆盖率的有损结构垄断了证据信道。**

## 1. 胜例机制解剖

### 1.1 fin_facts2 单元格表 —— 补【数值绑定】，加维不加量

- **事实**：美的等年报用"合并及公司"四栏双口径报表，扁平文本解析后列身份蒸发，
  模型面对数字汤无法分辨合并/母公司（fin_b_012 尸检，commit ed8cd10）。
  fin_b_012 是 full 线 13/14 错的顽固题，特护砸 21.3k 无效。
- **结构**：build_fin_facts2.py 用 fitz 词坐标 → y聚类成行 → 数字单元格x聚类成列 →
  表头绑定列身份（合并/公司 × 本期/上期），产出事实行。离线词法，零线上 token。
- **机制**：原始语料的缺陷不是"信息不在"而是"绑定维被解析破坏"。该结构不添加任何
  新内容，只把版面几何里本来就有的坐标信息恢复为显式绑定——**恰好补上 MSIS 缺失的
  那一个维度，过剩为零**。fin_b_012 史上首愈（finF），facts2 配方下 2/2 稳定治愈，
  router3 里 004+012 双愈。
- **对照**：full10 烧 4.59M（r1×3）仍治不了它——绑定缺失用采样预算不可替代。

### 1.2 insE 节选构卡 —— 压缩器输入先去噪，覆盖保障反而上升

- **事实**：全文构卡→15k 节选构卡（AFAC_WHOLE_LIMIT=15000），构卡账 350k→118k，
  ins 史上首个 20/20@585k，连永恒摇摆题 ins_b_010 也对了（台账注"节选卡去噪?"）。
- **机制**：卡片保真度是**压缩器输入信噪比**的函数，不是输入完整度的函数。
  保险条款的 MSIS（分档表/免责列举/犹豫期）密集在文档前部；全文输入让构卡模型
  在 5 倍噪声里做摘要，关键分档反而被稀释。节选=先用零成本的位置先验把 SNR 拉高，
  再让 LLM 压缩。砍掉的 85% 是压缩器的噪声，不是 MSIS——所以更便宜且更准。
- **可迁移表述**：压缩流水线的质量瓶颈常在输入端而非模型端；"喂全文求安心"是幻觉。

### 1.3 保险身份锚 + 全文记忆卡 —— 极小成本的【身份消歧】+【覆盖保障】

- **事实**：保险 PDF 标题层失真（doc9/10 同名、doc16 乱码），离线词法抽的
  公司+产品映射（insurance_titles.json）替换标题后，ins_b_004/008/017 类归属病根除，
  full 帧 ins 五连 19/20。A 榜全文记忆卡防"身故给付比例160%分档表"类条款漏检。
- **机制**：身份锚是**语料级**的绑定修复——几百字节的映射表校正了下游一切检索的
  归属坐标；错误不在题级而在库级，所以修一次全域受益。全文卡补的是覆盖保障：
  条款题的 MSIS 成员（某一档给付比例）事先不可预测，卡片按类目强制罗列
  （免责逐项不得概括，70e92e1）等于给覆盖上保险。
- **前置依赖警告**：同一张卡在 slim 瘦帽下 −4 键、大帽下 +11（slim15/24 三试盖棺）——
  覆盖型结构要占体积，帽不够时它挤掉的恰是别的题的 MSIS。**部件收益不可跨配置移植**。

### 1.4 tokenizer 年份归一 + 季度/方向同义表 —— 纯【表达对齐】，零内容坐标变换

- **事实**：题目写"2026年/一季度/下降"，文档写"26年/1-3月/同比-"。归一化后
  res_a_020 目标块从 >6 名升到第 4；分红 SYN 限域版（slim17）+3 键仅 +2.6k tok。
- **机制**：MSIS 百分之百在库里，输在查询面与文档面是两套方言。词表/tokenizer 层的
  对齐是**坐标变换**——不添加信息，只消灭表达鸿沟。这是全战役性价比最高的一类修复。
- **反面镜像**：slim16 把"全年"同义扩展开到全域 → 污染保险域查询，ins 13→6。
  对齐词表必须**挂域**：跨域的高频词映射=往别的域注入噪声（表达对齐做过头变成污染）。

### 1.5 批量摊薄 + facts2（finsB）—— 摊共享成本，保留逐题绑定

- **事实**：finsB（mix 家族批量帧 + facts2 速查行）以 **4,493 tok**（账实核：p3,936+c557）
  答对 fin_b_004=AB；同题在 full 帧 24k/题 的配置下 14 跑错 8 次（摇摆带）。
- **机制**：答题成本 = 共享部分（指令/判分标准/同域证据）+ 逐题边际（该题的绑定行）。
  批量把共享部分从每题一付改为一批一付；facts2 把 MSIS 载体从"原始表格页"换成
  "高绑定密度事实行"。两者叠加后不但便宜，还**更准**——干扰质量随上下文瘦身而下降，
  full 帧的错不是饿死而是 24k 杂讯里的判断摇摆，小而纯的上下文直接消灭了摇摆源。
- **要点**：摊薄的合法对象只有共享成本；逐题 MSIS 一个字不能摊（对照败例 1.4′ E1）。

### 1.6 JUDGE_STD 少样本判分校准 —— 输出端的【表达对齐】，零 token 键

- **事实**：规则13（口径词严格核算）让 fin_b_018 从 b_final 错到 full2 起 13 连对，
  同批法医修复 +5 键且 −200k；除字前缀/分红两笔/字面原句优先同理。
- **机制**：模型的"会答"与平台的"判对"之间也有表达鸿沟——答案空间的对齐和证据
  空间的对齐同构。少样本示例是把判分口径这个**隐藏绑定**显式化，成本为零、全域生效。

## 2. 败例机制解剖

### 2.1 finD 行级表替代宽检索（12/20@496k）—— 有损结构垄断信道

- **机制**：行级事实表 v1 的覆盖是 MSIS 的真子集（慢性四题的行不在表里），
  却**替代**了宽检索通道——缺的成员从此无法恢复，退化为闭卷。表本身没错，
  错在"未验证覆盖率就独占信道"。对照：同一张表以补丁形态进 full12 配方
  （router3）fin18 站住——**叠加+回退合法，替代需覆盖证明**。
- 深一层：行级粒度还丢了列身份（绑定不足），到单元格级 v2（facts2）才治病。
  即它同时踩了两条线：覆盖不足 + 绑定不足。

### 2.2 insH 卡片主导答题（13/20@219k）—— 概览分辨率 < 裁决分辨率

- **机制**：砍掉原文检索后，卡片成为唯一证据源。但判分口径要求字面原句优先，
  保险硬题的 MSIS 是**逐字条款**（分档数值/例外分句），卡片是有损概览——
  分辨率低于裁决所需。11k/题 的地板价买到的仍是"饿死"：不是钱不够，
  是载体的信息密度类型不对。卡片的正确角色=索引与覆盖保险丝，不是终审证据。

### 2.3 insI 条款速查表替代构卡（15/20@368k）—— schema 强加于开放材质

- **机制**：便宜 30% 丢 5 键。财报的 MSIS 天然是闭合表格 schema（科目×期间×口径
  有限枚举），所以 facts2 无损；保险条款是"叙述+表格+例外链"的**开放 schema**，
  预定义抽取字段等于在压缩时就截掉题目将会问到的长尾。同为"结构化速查表"，
  在 fin 是胜、在 ins 是败——**压缩形态必须匹配域的信息材质**，这是本战役最深的对称性证据。

### 2.4 E1 mega 上下文联合作答（fin 7/20@294k）—— 把逐题 MSIS 也摊没了

- **机制**：摊薄方向正确（同域 20 题一炉），但载体错了：900 字卡装不下年报表格密度，
  calc 题证据饿死。它验证了材质定律的另一半：卡片压缩对叙述知识高保真
  （reg 20/20@57k），对表格数值有损致命。共享化的只能是真共享的部分；
  每题各自的取数单元格是边际 MSIS，被"联合"进摘要即蒸发。
  （后续 finsB 用"批量+事实行"改对了载体，同一摊薄思想即告成功——死因在载体不在批量。）

### 2.5 r2 条件化（insF 19/20@545k、finE2 17/20@784k 双域证伪）—— 用自一致换独立性

- **机制**：验证的价值 = 第二意见错误分布的**独立性**，不是采样次数。
  r1 自一致是同模型相关采样——自信的错答案照样自一致，携带的独立信息趋零；
  跨代异构 r2（3.6主×3.5复）之所以是全战役最强单一武器（+5键零增账），
  正因为两代模型错误分布不同。r2 条件化把高信息量验证降级为低信息量自证，
  还为触发条件付出 r1 双票（194k→387k），信息量与成本双输。
  同族证据：r1×3 纯浪费（+1.26M 买 0 键）、slim23 异构治摇摆不治饿死
  ——判断层武器只在"证据在场"前提下有效，且效力排序=错误分布独立性排序。

## 4. 可迁移设计原理（五条）

1. **补维不加量**：先诊断缺的是哪类信息（绑定/身份/覆盖/对齐），用最小结构恰好补上
   那一维；用体积或采样预算替代缺失绑定必然失败。
   （fin_facts2 零 token 治 13/14 顽疾 vs full10 烧 4.59M 仍 90；身份锚几百字节根除归属病。）

2. **压缩形态必须匹配域的信息材质**：闭合 schema 材质（财报表格）用单元格级结构化，
   叙述材质（法规）用卡片，开放条款混合体（保险）只认原句+全文卡。
   （同一手法 facts2 在 fin 胜、速查表在 ins 败 15/20；卡片 reg 20/20@57k、fin 7/20 致命。）

3. **有损结构不得垄断信道**：未经覆盖率验证的压缩件只能以"叠加+原文回退"接入，
   替代原始通道需先证明其覆盖 ⊇ 域 MSIS。
   （finD 替代式 12/20 vs 同表补丁式 router3 fin18；insH 砍原文检索后 11k/题地板仍饿死。）

4. **验证的价值等于错误分布的独立性**：复核预算只该花在与主答错误不相关的信道上
   （跨代模型/异构证据面），同模型重采样与自一致检查近似零信息。
   （跨代异构 +5 键零增账；r2 条件化双域证伪；r1×3 +1.26M 买 0 键。）

5. **摊薄只对共享成本合法，且部件收益不可跨配置移植**：指令/判分标准/同域底仓可以
   一批一付，每题的边际 MSIS 一个字不能摊；任何部件的收益必须在目标预算配置里重验。
   （finsB 4.5k 解开 24k 解不开的题 vs E1 把逐题表格摊进 900 字卡 7/20；
   保险卡大帽 +11、瘦帽 −4。）

## 5. 什么样的知识结构才有力（80字）

有力的结构不是装更多信息，而是以最低成本把每题最小充分集连同绑定坐标完整送达：
数字绑住口径，证据绑住身份，表达对齐题面，压缩形态服从材质，有损件永留原文回退，
预算花在绑定与独立验证上。
