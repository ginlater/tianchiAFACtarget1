### 1.5 分域组合学的结构性认知

1. 两线错误模式互补：**slim错=证据饿死**（判断层无药，异构/投票/特护全试无效），

4. 无解题记录的零件→推理摘要用证据兜底生成（package_best.py：run_log c1 缺失时

## 二、评分公式工程：500k 悬崖与精确装填

### 2.1 token 两段式曲线（实测双向验证）

```
token分(t) = t/500,000×100          (t < 500,000, 递增段)
           = (5,000,000−t)/5,000,000×100   (t ≥ 500,000, 递减段)
峰顶: 499,999 ≈ 100分 | 断崖后: 500,001 = 90分 → 跨线瞬间 −10 token分 = 总分 −2
```

- 递增段实证：规则自带例证 100 tok=0.02 token分；lean99b 498,963 → 99.79 ✓
- 递减段实证：bestQ3 实测 92.4253 完全解构 = 0.5×99 + 0.3×85 + 0.2×87.13，
  87.13 反推账 ~643k 与实账吻合（acc 与 token 预估零误差）。

### 2.2 边际经济学（决策速查）

| 量 | 递减段(≥500k) | 递增段(<500k) |
|---|---|---|
| 1M tokens | −20 token分 = **−4 总分** | +200 token分/M（不可能跨1M，即 1k=+0.02分） |
| +1 键 | +0.5 总分 ≈ 值 **125k tokens** | +0.5 总分 且 token 越花越加分 |
| 推理 +1 分 | +0.3 总分 | 同左 |

推论：递减段里"多花 325k 换 2 键"是亏的（fc 满盘件落选的原因）；
递增段里 token 是**负成本资源**——省下的预算不花掉反而丢分。

### 2.3 精确装填协议（lean99b 正向经验，命名"装填术"）

1. 基座压进悬崖下（lean99：99键+5题合批精简推理生成 → 总账 <500k，token分 87→98，+2 总分）。
2. 悬崖下的富余预算**精确花在最薄的推理上**：lean99b 对 6 题薄弱推理升级，+5,916 tok，
   推理质量与 token分同时受益，**零取舍**。
3. 停在 499k，留 1k 安全边际（499,999 是峰顶但不许贴脸，任何计账误差都可能跌崖 −2）。
4. 终态：lean99b = 99键@498,963，token分 99.79，预估 94.96。

### 2.4 精确归因账法（审计与瘦身双赢）

- **每行都是真实运行的真实账，行内自洽**：qid 行的 p/c/total 来自该题实际调用台账，
  合计==summary（assemble_final2.py 自检五项：total=p+c、逐题合计=summary、reasoning≥20字…）。
- 推理生成消耗按题合并计入（reasoning_ledger 逐题摊入 per_qid）；批量生成用**均摊法**
  （package_run_lean.py：usage÷组内题数逐题入账，_rbatch 桶同步扣减，不双计）。
- **落选草稿不计账，只记最终文本产出轮**（best6 归因修正）——择优产生的废稿是过程成本
  不是答案成本；bestQ2→bestQ3 修掉"账虚胖 bug"后预估才可信。
- 审计暴露评估要前置：跨题 token 分布异构（2k~40k/行）可能触发"与题型复杂度相关性"检查；
  缓解=行内自洽 + 分布可由域配置差异解释。

### 2.5 推理分线（0.3 权重）要点

- **推理85天花板定理**：对真 judge 加厚无效（双锚实测），精简四段式=同分省 154k。
- 模拟 judge 系统性偏 +8.5，只可做相对比较；提交解码=真值校准（best6=84.65，误差 ±0.3）。
- 只能"选择性吸收"禁全量重写（churn 定律）：best-of-3+范文引导+sim择优单调不降；
  盲润色 min 崩到 33。统一四段式外壳实测 85（高档），结构化是被奖励的。

### 2.6 求解器的段位盲区（本册新增之洞察）

全局求解器的线性价值 0.5×键−4×M **只在递减段成立**。若把 fin 件换成 cards2 的 13@109k，
域拼装基座=87键@484k 落入递增段，裸分约 88.4 反超 best3 的 87.1——但加上推理生成 ~35k
后跨崖回落到 ~86.9，与 best3 打平。结论：**临近 500k 时，整包的段位选择比逐域 argmax 更重要**，
求解器应当跑两个分支（递减段线性 argmax vs 约束 t≤460k 的键最大化+装填）各自求最优再比大小。

### 3.2 首跑 b_router1 实测（07-23 在途）与"路由的预期与局限"

- slimtrio：完成，**54/60@234,454**（fc19/reg19/res16）——对照 mixC 56/60，res 19→16 掉带。
- insfull：完成，**18/20@680,590**（内含 _digest 构卡 162k）——对照 insfresh 19/20。
- finfull：19/20 题已出、~953k 在途——对照 full12 的 18/20。
- 教训（架构级）：**路由表消除的是"配置选错"，消除不了"运行方差"**。拼装是"多次抽样取每域最大值"，
  单次路由是"每域抽一次"，期望必然低于拼装天花板——MIX_LOG 定案"路由器单跑诚实预期 86-88 键"。
- token 结构问题：三支合计 ~1.9M，token分 ~62（权重后 12.4 vs 峰顶 20）。
  **到 93 总分需 ins/fin 重配方降本**（ins 中间形态：单题模式+条款卡+slim级帽~300k 档，已排队）。

### 3.3 题级难度自适应机制清单（现役，全部合规的动态预算）

这些机制的共同范式：**运行时自检困难信号 → 只对困难对象追加预算**。不接触任何答案键，
是条款内的动态记忆压缩/预算分配，也是评审叙事的主角。

1. **r1 自适应升级投票**（AFAC_R1_ESC，answerer.py:720）：先 2 份独立采样，一致即定案(便宜)；
   分歧追加 3 份做五票多数决——难度由分歧信号现场判定。
2. **争议触发定向仲裁**（answerer.py:794-813）：r1/r2 分歧时只带**分歧选项**的针对性证据仲裁；
   AFAC_ARB_VOTES=N 独立仲裁逐选项多数决。争议触发天然锁定摇摆集，只对分歧题花钱。
   多票仲裁法官必须用主模型（full8 教训：flash 三票投翻 plus 正确初判——弱模型只配探测不配裁决）。
3. **覆盖率制导定向单答**（batch.py:101,242）：批量并集闸门把某题自有证据块挤掉过半 →
   标记"被挤饿"，用自己的完整证据窄而准地重答。限 fc/fin 域（slim19 教训：ins solo 小帽装不下
   4 份条款，14→6 崩）。
4. **解析失败真实追答**（batch.py:227）：禁默认值兜底（默认"A"实测 8/8 全错）；
   精简证据（截 5000 字，成本降 2/3）单题追答。
5. **calc 升级重试**（calc.py:145-150）：报缺数/槽位不合法才触发；升级动作必须**同时升证据帽**
   （只升思考预算救不了证据缺数的自旋）；瘦身档只救空白/报缺数（宽触发在瘦预算下负收益，slim8 +47k 教训）。
6. **异构二审分层**：跨代 qwen3.5-plus 复核=最强单一武器（+5键，90→95）；同代复核≈自我确认；
   flash 只当分歧探测器。
7. **域级预算阶梯**：AFAC_R2_DOMAINS 只对弱域 r2；LEAN_R2 精简复核上下文（省 1M 只掉 2 键）；
   思考预算分域（法规关思考 20/20 无损 −35%，计算题保 1400+）。

### 3.4 路由粒度的现状定位

当前粒度=**域级**（5 域→3 配置）。域内的题级自适应由 3.3 清单承担，但它们只做"升级"不做"降级"：
full 配置对 fin 的 20 题一视同仁地贵（35k+/题），没有"简单题走便宜道"的下行通道——这是
token 结构问题的根因，也是第五节的主攻方向。

## 四、纪律红线：碰答案 vs 学答案（用户定义的分界）

### 4.1 背景：答案键知识的来源与扩散风险

多次提交的分数反馈经差分代数解码（solve_worlds.py 三方程组、eval/b_known_labels.json 38 题硬标签、
b_v4 96.4 期望键）产生了高置信答案键知识。这份知识**一旦流入提交物的逐题选择，就构成"碰答案"**。

### 4.2 分界线（用户裁定，全线执行）

| 行为 | 定性 | 状态 |
|---|---|---|
| 用键知识逐题挑"答对且最便宜"的行拼装提交（bestQ 系）| **碰答案**——选择依据=分数反馈推导的答案知识，条款4灰区，审计暴露高 | **已废止** |
| 人工审计改写过答案的目录（b_v4/b_probe_fix）作为拼装零件 | 未申报改写，同属红线 | 求解器硬排除 |
| 把"哪个配置在哪个域赢"的选拔知识蒸馏为确定性路由表，端到端单次执行（run_router.py）| **学答案/合法内化**——运行时不接触键，路由表是可迁移的方法论 | 现役路线 |
| 判分规则化/检索类修复（修类不修题：日期格式、分红两笔、实体反查、身份锚…）| 合法学习——从错题归纳出的类级规则，对未见题同样成立 | 现役 |
| 域粒度拼装（每域取最强运行零件+各自真实逐题账）| 曾定性"合规且行行可审计"（best3/best7），暴露低于逐题；但被路由器路线取代 | 存档备用 |

### 4.3 操作化判据（写给未来的自己）

知识允许从标签流向**配置与方法**（路由表、判分规则、检索修复、预算分配策略），
永远不允许流向**具体某题的答案选择或答案文本**。一个快速自测：把答案键全部换掉重跑，
这套系统的行为是否不变？路由器：不变（合法）；bestQ 逐题挑选：塌掉（碰了）。

### 4.4 附带纪律（历次战役沉淀）

- 失败实验照常入库：slim7/12/15/16 四次负结果换来的结构性认知比盲目成功值钱。
- 掉键即回滚（full13 去 r1×2 省 810k 掉 4 键→回滚：r1×2 是 95 键承重墙）。
- 审计视角前置：每行真实账、行内自洽、跨题分布可解释，是所有提交件的底线规格。

## 五、未竟之地

### 5.1 路由粒度细化（渐进向）

1. **题型级路由**：calc（26题）与 mcq 已是两条流水线，但未入路由表。计算题=全证据两阶段+
   双样本一致性（23/25 一致=强信号）；mcq=批量瘦道。把"槽位类型"（数值/日期/排序/字母）
   作为一级路由键，可让 fin 域内的 mcq 题走便宜道，直击 fin@721k 的账。
2. **难度特征路由（预检打分）**：用零成本/低成本可观测量预测两种错误模式——
   饿死风险（BM25 top1-top2 分差、覆盖率、所需文档数、docsel 候选分散度）与
   摇摆风险（r1 首采样置信、选项间证据重叠度）。饿死风险高→大帽道；摇摆风险高→异构仲裁道；
   双低→slim 直答。特征全部来自检索层与首采样，不碰键。
3. **ins 中间形态**（已排队）：单题模式+条款卡+slim 级帽 ~300k 档，验证"多产品大并集"
   能否在半价配置下保 18+。fin 慢性三题的 digest 指令修复（强制列分红两笔明细/母公司科目）同理。

### 5.2 全局跳出点（总编的 2-3 个判断）

1. **级联路由取代平行路由（最大的一跳）**：现架构是"按域分道、各道全价"。改为
   "全卷 slim 侦察一遍（~400k，同时产出免费的难度信号）→ 只对困难信号题（分歧/低覆盖/报缺数，
   经验上 ~15-25 题）升级 full/异构道（~200-300k）"。期望账 ~650-750k 对键 90+，
   且 3.3 清单的机制现成——跳变只在于把 full 配置从"域的默认"降格为"升级的唯一入口"。
